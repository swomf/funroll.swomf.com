Now, a thought experiment:

Imagine you keep stubbing your foot on AI-enabled Windows Explorer, or the Documents/ folder that is actually OneDrive, or the magic nonsense you fired into the Windows Registry. How 86ca1aa0-34aa-4e8b-a509-50c905bae2a2-esque.

Imagine you use Linux instead. Yay! Satisfaction with a system that actually likes you. Except you sometimes still stub your toe on peculiar design decisions. You fix it but forget it later on. Are your toes safe?

Imagine you then jump to Arch Linux. Less things to stub your toe here. After some literacy and time, you set it up once, and boom, rewards:

  1. A lord-tier wiki where you're the first-class citizen
  2. Download-and-install packages that are mere text files
  3. Oops! Selective updates and downgrades are unsupported.

If you have a life, you set up pre-update backups or jump to Fedora. Probably. Unless you get dangerous ideas about how the system should work. That the package manager should track what you season its packages with.

Imagine you go nuts and pick the guarantees of NixOS or the assisted-Linux-from-source of Gentoo. Or maybe BSD.

The source code has never been easier to slap a hot rod engine onto. Now the easy stuff is harder, the hard stuff is easier, and the impossible stuff is only hard. Ground hurts your feet? Recompile it.

Imagine in particular: you follow me into Gentoo-land. Now we can

  • Change install-time supports by USE flags in package.use
  • Change install-time environment variables by package.env
  • Choose between stable, unstable, and bleeding-edge by package.ACCEPT_KEYWORDS
  • Handmake patches/use others' patches via patches
  • Add little changes to install recipes via install hooks

Caveat. Those are all in different directories. What if, for some grand plan, you change three things about one package and also change one thing about three packages? How do you remember to keep these changes synced?

It would be nice if you had a bunch of Markdown files explaining the code blocks you added.

It would be nicer if you could automatically rip out the code blocks right into your system config files so that the documentation syncs with the state.

It would be nicest if this became a pretty website at the same time.

Congratulations! We invented funroll. And as Donald Knuth envisioned, we have a "literate programming" architecture. (Wikipedia ⇗)
